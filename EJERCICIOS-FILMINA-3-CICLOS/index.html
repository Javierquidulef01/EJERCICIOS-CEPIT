EJERCICIOS PRÁCTICOS: CICLOS

A continuación se presentan los ejercicios de iteración correspondientes a la Presentacion "3. Ciclos" utilizando HTML y TYPESCRIPT.
En estos ejercicios se hace uso de las estructuras de control While y For, y de las funciones Prompt y Alert.

EJERCICIO 1: EUREKA

Algoritmo que pide una clave y verifica si es la correcta. Solamente se permitirán 3 intentos para acertar.
Si acertamos, se terminará el programa. Y si fallamos, se nos mostrará un mensaje indicando que se agotaron los intentos.
La clave correcta es la palabra "eureka".
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.



EJERCICIO 2: MÚLTIPLOS

Algoritmo que muestra los números que son múltiplos de 2 ó de 3 y que se encuentran entre el 1 y el 100.
A los números que son múltiplos de 2 y de 3 al mismo tiempo, se los mostrará por una única vez.
El informe de los números se realizará por consola.



EJERCICIO 3: PAR/IMPAR

Algoritmo que dado un número entero ingresado por el usuario, nos informará en pantalla si es PAR o IMPAR.
En el caso de ingresar un 0 (CERO), nos volverá a pedir el ingreso de un número hasta que éste sea distinto de cero.
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.



EJERCICIO 4: SUMA ENTRE NÚMEROS

Algoritmo que no pide dos números enteros, y luego retorna la suma de todos los números que están entre ellos.
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.
Ejemplo: inmput: 2 y 5; output: 14 (= 2 + 3 + 4 + 5)



EJERCICIO 5: TABLAS DE MULTIPLICACIÓN

Algoritmo que muestra la tabla de multiplicación del número ingresado por el usuario.
Además se deberá ingresará un número que definirá hasta que número se desea que muestre la tabla de multiplicación.
La entrada de datos se realizará mediante la función PROMPT y la salida por CONSOLA.



EJERCICIO 6: ENCONTRAR EL NÚMERO MÁXIMO

Algoritmo que lee numeros (positivos o negativos) hasta que se introduzca un 0 (CERO).
De todos los números ingresados, el algoritmo muestra el numero maximo ingresado.
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.
Además se imprimirán por consola todos los números ingresados.



EJERCICIO 7: ENCONTRAR EL NÚMERO MÍNIMO

Algoritmo inverso al del ejercicio 6.



EJERCICIO 8: CANTIDAD Y DISTRIBUCIÓN DE POSITIVOS

Algoritmo que lee números enteros hasta que se introduzca un 0 (CERO) y muestra la cantidad de
números positivos que fueron ingresados y el procentaje de positivos respecto del total.
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.



EJERCICIO 9: PROMEDIO, MÁXIMO Y MÍNIMO

Algoritmo que lee números enteros hasta que se introduzca un 0 (CERO) y además, Determina y muestra
el máximo, el mínimo y la media de todos los números ingresados
La entrada de datos se realizará mediante la función PROMPT y la salida por CONSOLA.



EJERCICIO 10: CALIFICACIONES

Algoritmo que calcula las calificaciones de un grupo de alumnos, donde la nota final de cada alumno se calcula según el siguiente criterio:
• la parte práctica vale el 10%
• la parte de problemas vale el 50%
• la parte teórica el 40%
Se debe ingresar el nombre del alumno y sus tres notas, se escribirá el resultado y se volverá a pedir los datos
del siguiente alumno hasta que el nombre sea una cadena vacía.
Las notas deben estar entre 0 y 10 (si no lo están, no imprimirá las notas, mostrará un mensaje de error y continuará con otro alumno)
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.



EJERCICIO 11: DADOS

Programa que calcula la probabilidad de sacar todos los dados 6 siendo que tiramos N dados (dato ingresado por al usuario)
Ejemplo:
• Al tirar un dado tenemos 1/6 de probabilidades de sacar 6.
• Si tiramos dos dados tenemos 1/36 probabilidades de sacar doble 6.
La entrada de datos se realizará mediante la función PROMPT y la salida mediante la función ALERT.
